## Tech Stack

- **Framework**: TanStack Start (React 19) with file-based routing
- **UI**: shadcn/ui using **base-nova** style (`@base-ui/react` primitives, NOT Radix — no `asChild` prop, use `render` prop instead)
- **Styling**: Tailwind CSS v4 with `@tailwindcss/vite` plugin
- **Deployment**: Vercel via Nitro (`nitro/vite` plugin in vite.config.ts)
- **Import alias**: `#/` maps to `./src/*` (defined in both package.json `imports` and tsconfig `paths`)

## Architecture

Motify is a background pattern generator with a **layered engine** that composes CSS backgrounds from independent layers.

### Engine (`src/engine/`) — Pure TypeScript, zero React deps

The engine converts a `PresetConfig` object into CSS `background-image` / `background-size` strings by stacking three layer types:

1. **Base** — solid color or gradient (`BaseLayer`)
2. **Motif** — repeating geometric pattern (`MotifLayer`) generated by pluggable functions
3. **Modifiers** — optional overlays like vignette/fade (`ModifierLayer[]`)

**Motif registry** (`src/engine/motifs/index.ts`): Maps `MotifType` → generator function. Each generator has the signature `(motif: MotifLayer) => { backgroundImage, backgroundSize }`. To add a new pattern, create a generator file and register it.

**CSS composition** (`src/engine/css-generator.ts`): Stacks all layers into comma-separated `background-image` values. Critical detail: `backgroundSize` must have one entry per `backgroundImage` layer — use `countTopLevelCommas()` to count motif sublayers.

**Colors**: Stored as hex strings (`#RRGGBB`). Converted to rgba via `withOpacity()` in `src/engine/utils.ts`.

### UI Flow

`/` (Gallery) → click preset → `/editor?preset={id}` → `useState<PresetConfig>` at page level → `useMemo(() => generateCSS(config))` → inline styles on preview div. All parameter changes update the single config state object.

### Key Patterns

- shadcn components live in `src/components/ui/` and use `@base-ui/react` (not Radix)
- Shared components (color picker, pattern preview) in `src/components/shared/`
- Feature components split into `src/components/gallery/` and `src/components/editor/`
- Presets are JSON config objects in `src/engine/presets.ts`, not hardcoded CSS
- Tailwind color palette data is in `src/lib/tailwind-colors.ts`
